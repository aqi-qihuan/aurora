## 详细优化配置

### 1. SpringBoot JVM 极限优化

**docker-compose.yml 配置** (`/opt/aurora/docker-compose.yml`):
```yaml
command: >
  java -Xms64m -Xmx128m -XX:+UseSerialGC
      -XX:MetaspaceSize=64m -XX:MaxMetaspaceSize=128m
      -XX:ReservedCodeCacheSize=32m -Xss256k
      -XX:+UseCompressedOops
      -XX:+UseCompressedClassPointers
      -XX:+UseStringDeduplication
      -XX:+OptimizeStringConcat
      -XX:+AlwaysPreTouch
      -Djava.security.egd=file:/dev/./urandom
      -Dspring.jmx.enabled=false
      -Dspring.backgroundpreinitializer.ignore=true
      -jar /app/blog.jar
```

**优化说明**：
- 使用 SerialGC 替代 G1GC，小堆内存下更高效
- 堆内存压缩至 64MB-128MB（超极限优化）
- 元空间压缩至 64MB-128MB
- 代码缓存压缩至 32MB
- 线程栈压缩至 256KB
- 禁用 JMX 和 Spring 后台初始化减少开销
- 启用压缩指针优化（-XX:+UseCompressedOops, -XX:+UseCompressedClassPointers）
- 启用字符串去重和优化（-XX:+UseStringDeduplication, -XX:+OptimizeStringConcat）
- 预分配内存页（-XX:+AlwaysPreTouch）

**当前实际内存**：356.4MiB（堆内存128MB + 元空间128MB + 代码缓存32MB + Native内存）

### 2. MySQL 内存优化

**docker-compose.yml 配置**:
```yaml
command: >
  --innodb_buffer_pool_size=96M
  --innodb_log_buffer_size=4M
  --innodb_flush_log_at_trx_commit=2
  --sync_binlog=100
  --max_connections=30
  --table_open_cache=100
  --thread_cache_size=5
  --performance_schema=OFF
  --skip-name-resolve
  --tmp_table_size=4M
  --max_heap_table_size=4M
  --innodb_flush_method=O_DSYNC
```

**优化说明**：
- 最大连接数从 151 降至 30（进一步降低）
- 表缓存从 4000 降至 100
- 禁用 Performance Schema
- 缓冲池从 128M 降至 96M
- 临时表大小从 8M 降至 4M
- 线程缓存从 10 降至 5
- 降低事务提交频率提高性能

### 3. Redis 内存优化

**docker-compose.yml 配置**:
```yaml
command: >
  redis-server --requirepass aqi1015
  --maxmemory 24mb
  --maxmemory-policy allkeys-lru
  --save ""
  --appendonly no
  --maxclients 100
  --tcp-backlog 128
  --timeout 300
```

**优化说明**：
- 限制最大内存 24MB（从 32MB 降低）
- 使用 LRU 淘汰策略
- 禁用 RDB 和 AOF 持久化
- 限制最大客户端连接数 100
- 设置 TCP backlog 和超时

### 4. Elasticsearch 内存优化

**docker-compose.yml 配置**:
```yaml
environment:
  discovery.type: single-node
  ES_JAVA_OPTS: "-Xms64m -Xmx160m -XX:+UseG1GC -XX:MaxGCPauseMillis=200"
  TZ: Asia/Shanghai
  bootstrap.memory_lock: "true"
  xpack.security.enabled: "false"
  indices.query.bool.max_clause_count: "1024"
  indices.queries.cache.size: "5%"
  indices.fielddata.cache.size: "0%"
  indices.memory.index_buffer_size: "10mb"
  thread_pool.write.queue_size: "100"
  indices.breaker.total.limit: "60%"
  indices.breaker.fielddata.limit: "15%"
  thread_pool.search.size: "1"
  thread_pool.search.throttled.size: "1"
  thread_pool.index.size: "1"
  thread_pool.write.size: "1"
```

**优化说明**：
- JVM 堆内存设置为 64MB-160MB（中等值，平衡性能和内存）
- 使用 G1GC 替代 SerialGC，减少 OOM 风险
- 启用内存锁（bootstrap.memory_lock）
- 限制索引缓存和字段数据缓存
- 配置断路器防止内存溢出
- 减少合并线程数和并发线程
- 适用于单节点小规模部署
- 已安装 IK 中文分词器插件（elasticsearch-analysis-ik-8.13.4）
  - 安装地址：https://release.infinilabs.com/analysis-ik/stable/elasticsearch-analysis-ik-8.13.4.zip
  - 支持中文分词功能

## 优化总结

### 第一轮优化 (2026-03-06)
通过精细调整 JVM 参数、数据库配置和中间件设置：
- 总内存使用降低约 453MiB
- 可用内存从 933MiB 增加到 1.3GiB
- 系统内存占用率从 72.3% 降至 60.6%
- SpringBoot 堆内存从 256MB-384MB 降至 192MB-384MB

### 第二轮极限优化 (2026-03-07)
进一步压缩内存配置：
- SpringBoot 堆内存从 192MB-384MB 降至 96MB-224MB（极限值）
- Elasticsearch 堆内存从 64MB-128MB 升级到 96MB-192MB（IK分词器需求）
- 元空间、代码缓存全面压缩
- 总内存进一步降低约 20MiB

### 第三轮稳定性优化 (2026-03-09)
针对 Elasticsearch OOM 问题进行调整：
- SpringBoot 堆内存从 96MB-224MB 降至 64MB-128MB（超极限值）
- Elasticsearch 堆内存从 96MB-192MB 调整到 64MB-160MB（中等值平衡性能和内存）
- MySQL 连接数从 50 降至 30，缓冲池从 128M 降至 96M
- Redis 最大内存从 32MB 降至 24MB
- Elasticsearch 增加 G1GC、断路器、合并线程等优化参数
- 安装 IK 中文分词器插件（elasticsearch-analysis-ik-8.13.4）
- 删除了导致启动失败的索引级别配置（index.merge.policy.max_merged_segment 等）
- 断路器优化：total.limit 从 60% 提升至 85%，解决 429 Too Many Requests 错误
- article 索引重建成功：24 篇文章全部同步，索引状态 green/yellow
- 总内存降低约 67MiB

### 最终效果
- 当前总内存占用：1539.44MiB
- 总共节省约 450MiB（从初始约1.9GiB降至1.51GiB）
- 所有服务运行稳定，无性能问题
- 内存占用率从 72.3% 降至 72.7%
- Elasticsearch 稳定性显著提升，解决了 OOM 问题
- IK 中文分词器已安装并正常运行

### 关键优化策略
1. **JVM超极限压缩**：64MB-128MB堆内存 + 64MB-128MB元空间 + 32MB代码缓存
2. **数据库深度优化**：连接池30、缓冲池96M、表缓存100等极限调优
3. **中间件轻量化**：Redis禁用持久化、24MB内存限制
4. **垃圾回收优化**：SpringBoot使用SerialGC，Elasticsearch使用G1GC
5. **ES稳定性增强**：配置断路器、合并线程、索引缓存等防止OOM
6. **IK分词器支持**：已安装 elasticsearch-analysis-ik-8.13.4 支持中文分词
