## 🖥️ 1. 环境准备

### 1.1 服务器要求

#### 推荐配置

| 配置 | ES搜索策略 | MySQL搜索策略 |
|------|-----------|---------------|
| CPU | 2核 | 2核 |
| 内存 | 4GB | 2GB |
| 带宽 | 1M+OSS 或 3M+MinIO | 1M+OSS 或 3M+MinIO |

#### 内存优化效果

通过极限优化配置（详见[内存优化总结](./内存优化总结.md)），在2核4G服务器上的实际内存占用：

| 组件 | HotSpot JVM | GraalVM JIT | 占比 | 优化配置 |
|------|:----------:|:----------:|------|---------|
| SpringBoot | 342.8MiB | **275.9MiB** | 18.7% | GraalVM JDK 25, -Xms64m -Xmx128m + SerialGC |
| Elasticsearch | 652.9MiB | 652.9MiB | 44.4% | -Xms64m -Xmx160m + G1GC + IK分词器 |
| MySQL | 148.3MiB | 148.3MiB | 10.1% | 连接池30、缓冲池96M |
| Maxwell | 150.4MiB | 150.4MiB | 10.2% | 自动优化 |
| MinIO | 130.1MiB | 130.1MiB | 8.8% | 缓存优化 |
| RabbitMQ | 107.8MiB | 107.8MiB | 7.3% | 默认配置 |
| Redis | 4.0MiB | 4.0MiB | 0.3% | 24MB内存限制 |
| Nginx | 3.1MiB | 3.1MiB | 0.2% | 默认配置 |
| **总计** | **1539.44MiB** | **1472.55MiB** | **100%** | **内存占用率约69.5%** |

> 💡 **GraalVM 优势**：SpringBoot 内存降低 **-19.5%**（342.8MiB → 275.9MiB），总内存节省约 **66.9MiB**

> **注意**：如果不需要MinIO，可节省约130MB内存。

### 1.2 🐳 Docker安装

#### Ubuntu系统

```shell
# 更新软件包索引
apt-get update

# 安装依赖包
apt-get install -y ca-certificates curl gnupg lsb-release

# 添加Docker官方GPG密钥
curl -fsSL https://mirrors.aliyun.com/docker-ce/linux/ubuntu/gpg | gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# 设置稳定版仓库
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://mirrors.aliyun.com/docker-ce/linux/ubuntu $(lsb_release -cs) stable" | tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装Docker Engine
apt-get update
apt-get install -y docker-ce docker-ce-cli containerd.io

# 启动Docker并设置开机自启
systemctl start docker
docker version
systemctl enable docker
```

## 📦 2. 中间件部署

使用docker-compose统一部署所有中间件，方便管理和维护。

### 2.1 创建项目目录

```shell
mkdir -p /opt/aurora
cd /opt/aurora
```

### 2.2 创建docker-compose.yml

在`/opt/aurora/`目录下创建`docker-compose.yml`：

```yaml
version: "3.8"
services:
  mysql:
    image: mysql:8.0.32
    container_name: aurora-mysql
    environment:
      MYSQL_ROOT_PASSWORD: 你的密码
      MYSQL_DATABASE: aurora
      TZ: Asia/Shanghai
    ports:
      - "3306:3306"
    volumes:
      - /opt/aurora/mysql/data:/var/lib/mysql
    networks:
      - aurora-network
    restart: always
    command: >
      --innodb_buffer_pool_size=96M
      --innodb_log_buffer_size=4M
      --innodb_flush_log_at_trx_commit=2
      --sync_binlog=100
      --max_connections=30
      --table_open_cache=100
      --thread_cache_size=5
      --performance_schema=OFF
      --skip-name-resolve
      --tmp_table_size=4M
      --max_heap_table_size=4M
      --innodb_flush_method=O_DSYNC

  redis:
    image: redis:7.0.8
    container_name: aurora-redis
    ports:
      - "6379:6379"
    volumes:
      - /opt/aurora/redis/data:/data
    networks:
      - aurora-network
    restart: always
    command: >
      redis-server --requirepass 你的密码
        --maxmemory 24mb
        --maxmemory-policy allkeys-lru
        --save ""
        --appendonly no
        --maxclients 100
        --tcp-backlog 128
        --timeout 300

  rabbitmq:
    image: rabbitmq:3.11.9-management
    container_name: aurora-rabbitmq
    environment:
      RABBITMQ_DEFAULT_USER: myroot
      RABBITMQ_DEFAULT_PASS: 你的密码
      TZ: Asia/Shanghai
    ports:
      - "5672:5672"
      - "15672:15672"
    volumes:
      - /opt/aurora/rabbitmq/data:/var/lib/rabbitmq
    networks:
      - aurora-network
    restart: always

  elasticsearch:
    image: elasticsearch:8.13.4
    container_name: aurora-elasticsearch
    environment:
      discovery.type: single-node
      ES_JAVA_OPTS: "-Xms64m -Xmx160m -XX:+UseG1GC -XX:MaxGCPauseMillis=200"
      TZ: Asia/Shanghai
      bootstrap.memory_lock: "true"
      xpack.security.enabled: "false"
      indices.query.bool.max_clause_count: "1024"
      indices.queries.cache.size: "5%"
      indices.fielddata.cache.size: "0%"
      indices.memory.index_buffer_size: "10mb"
      thread_pool.write.queue_size: "100"
      indices.breaker.total.limit: "85%"
      indices.breaker.fielddata.limit: "10%"
      thread_pool.search.size: "1"
      thread_pool.search.throttled.size: "1"
      thread_pool.index.size: "1"
      thread_pool.write.size: "1"
    ports:
      - "9200:9200"
      - "9300:9300"
    volumes:
      - /opt/aurora/elasticsearch/data:/usr/share/elasticsearch/data
    networks:
      - aurora-network
    restart: always
    ulimits:
      memlock:
        soft: -1
        hard: -1

  minio:
    image: bitnami/minio:2023.12.7
    container_name: aurora-minio
    hostname: minio-server
    environment:
      MINIO_ROOT_USER: "你的用户名."
      MINIO_ROOT_PASSWORD: "你的密码."
      MINIO_DEFAULT_BUCKETS: "aurora"
      MINIO_SERVER_URL: "http://127.0.0.1:9000"
      TZ: Asia/Shanghai
      MINIO_CACHE_DRIVES: /tmp/cache
      MINIO_CACHE_EXPIRY: 90
    ports:
      - "9000:9000"
      - "9001:9001"
    volumes:
      - /opt/aurora/minio/data:/bitnami/minio/data
    networks:
      - aurora-network
    restart: always

  nginx:
    image: nginx:1.23.3
    container_name: aurora-nginx
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - /opt/aurora/nginx/conf:/etc/nginx
      - /opt/aurora/nginx/html:/usr/share/nginx/html
      - /opt/aurora/vue/blog:/usr/share/nginx/blog
      - /opt/aurora/vue/admin:/usr/share/nginx/admin
      - /opt/aurora/nginx/cert:/etc/nginx/cert
    networks:
      - aurora-network
    restart: always

  springboot:
    image: eclipse-temurin:25-jre-alpine
    container_name: aurora-springboot
    environment:
      SPRING_PROFILES_ACTIVE: prod
      TZ: Asia/Shanghai
    ports:
      - "8080:8080"
    volumes:
      - /opt/aurora/app/aurora-springboot-0.0.1.jar:/app/blog.jar
    networks:
      - aurora-network
    restart: always
    command: >
      java -Xms64m -Xmx128m -XX:+UseSerialGC
          -XX:MetaspaceSize=64m -XX:MaxMetaspaceSize=128m
          -XX:ReservedCodeCacheSize=32m -Xss256k
          -XX:+UseCompressedOops
          -XX:+UseCompressedClassPointers
          -XX:+UseStringDeduplication
          -XX:+OptimizeStringConcat
          -XX:+AlwaysPreTouch
          -Djava.security.egd=file:/dev/./urandom
          -Dspring.jmx.enabled=false
          -Dspring.backgroundpreinitializer.ignore=true
          -jar /app/blog.jar

  maxwell:
    image: zendesk/maxwell:latest
    container_name: aurora-maxwell
    volumes:
      - /opt/aurora/maxwell/config/config.properties:/app/config.properties
    networks:
      - aurora-network
    restart: always
    command: >
      bin/maxwell --host=aurora-mysql --user=root --password=你的密码
        --producer=rabbitmq --rabbitmq_host=aurora-rabbitmq
        --rabbitmq_user=myroot --rabbitmq_pass=你的密码
        --rabbitmq_exchange=maxwell --rabbitmq_exchange_type=topic
        --rabbitmq_routing_key_template=%{database}.%{table}
        --client_id=maxwell_prod

networks:
  aurora-network:
    external: true
```

**重要提示**：请将配置中的`你的密码`、`你的用户名`替换为实际密码。

### 2.3 🐘 GraalVM 版本配置（可选）

如果需要使用 GraalVM JIT 编译器替代 HotSpot JVM（性能提升 10-20%，内存节省约 19.5%），可以使用以下配置：

```yaml
# GraalVM 版本 - 将上述 docker-compose.yml 中的 springboot 服务替换为：
  springboot:
    image: ghcr.io/graalvm/jdk-community:25
    container_name: aurora-springboot
    environment:
      SPRING_PROFILES_ACTIVE: prod
      TZ: Asia/Shanghai
    ports:
      - "8080:8080"
    volumes:
      - /opt/aurora/app/aurora-springboot-0.0.1.jar:/app/blog.jar
    networks:
      - aurora-network
    restart: always
    command: >
      java -Xms64m -Xmx128m -XX:+UseSerialGC
          -XX:MetaspaceSize=64m -XX:MaxMetaspaceSize=128m
          -XX:ReservedCodeCacheSize=32m -Xss256k
          -XX:+UseCompressedOops
          -XX:+UseCompressedClassPointers
          -XX:+UseStringDeduplication
          -XX:+OptimizeStringConcat
          -XX:+AlwaysPreTouch
          -Djava.security.egd=file:/dev/./urandom
          -Dspring.jmx.enabled=false
          -jar /app/blog.jar
```

> ⚠️ **注意**：GraalVM 官方没有 Alpine 版本，镜像约 500MB（比 Alpine 大约 400MB）。需要运行 10-30 分钟才能体现 JIT 优化效果。

```shell
cd /opt/aurora
docker-compose up -d
```

### 2.5 📊 查看服务状态

## ⚙️ 3. 配置初始化

### 3.1 🔍 安装IK分词器

进入Elasticsearch容器：

```shell
docker exec -it aurora-elasticsearch /bin/bash
```

安装IK分词器：

```shell
./bin/elasticsearch-plugin install https://release.infinilabs.com/analysis-ik/stable/elasticsearch-analysis-ik-8.13.4.zip
```

> **注意**：IK分词器版本必须与Elasticsearch版本一致（8.13.4）

退出容器并重启：

```shell
exit
docker restart aurora-elasticsearch
```

验证安装：

```shell
docker exec aurora-elasticsearch elasticsearch-plugin list
```

测试分词功能：

```shell
curl -X POST 'http://服务器IP:9200/_analyze' \
  -H 'Content-Type: application/json' \
  -d '{
    "tokenizer": "ik_smart",
    "text": "我爱技术"
  }'
```

### 3.2 🌐 创建外部网络

如果网络不存在，需先创建：

```shell
docker network create aurora_aurora-network
```

## ☕ 4. 后端部署

### 4.1 修改配置文件

修改`aurora-springboot/src/main/resources/application-prod.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://aurora-mysql:3306/aurora?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=GMT%2B8
    username: root
    password: 你的密码

  redis:
    host: aurora-redis
    port: 6379
    password: 你的密码

  rabbitmq:
    host: aurora-rabbitmq
    port: 5672
    username: myroot
    password: 你的密码

elasticsearch:
  host: aurora-elasticsearch
  port: 9200
  # 已禁用安全认证，无需用户名密码
```

### 4.2 第三方服务配置

**阿里云OSS**：
- 在阿里云控制台创建OSS bucket
- 获取AccessKey和SecretKey
- 在配置文件中填写相关参数

**QQ互联**：
- 在QQ互联平台申请应用
- 获取APP_ID和APP_KEY
- 配置回调地址

### 4.3 打包应用

```shell
cd aurora-springboot
mvn clean package -DskipTests
```

### 4.4 上传jar包

```shell
mkdir -p /opt/aurora/app
```

将`aurora-springboot-0.0.1.jar`上传到`/opt/aurora/app/`目录。

### 4.5 创建Dockerfile

在`/opt/aurora/app/`目录下创建`Dockerfile`：

**HotSpot JVM 版本（默认）**：
```dockerfile
FROM eclipse-temurin:25-jre-alpine
VOLUME /tmp
ADD aurora-springboot-0.0.1.jar blog.jar

# JVM 极限优化参数
ENTRYPOINT ["java", "-Xms64m", "-Xmx128m", "-XX:+UseSerialGC",
  "-XX:MetaspaceSize=64m", "-XX:MaxMetaspaceSize=128m",
  "-XX:ReservedCodeCacheSize=32m", "-Xss256k",
  "-XX:+UseCompressedOops", "-XX:+UseCompressedClassPointers",
  "-XX:+UseStringDeduplication", "-XX:+OptimizeStringConcat",
  "-XX:+AlwaysPreTouch", "-Djava.security.egd=file:/dev/./urandom",
  "-Dspring.jmx.enabled=false", "-Dspring.backgroundpreinitializer.ignore=true",
  "-jar", "/blog.jar"]
```

**GraalVM 版本（可选，性能更优）**：
```dockerfile
FROM ghcr.io/graalvm/jdk-community:25
VOLUME /tmp
ADD aurora-springboot-0.0.1.jar blog.jar

# JVM 极限优化参数 + Graal JIT 编译器
ENTRYPOINT ["java", "-Xms64m", "-Xmx128m", "-XX:+UseSerialGC",
  "-XX:MetaspaceSize=64m", "-XX:MaxMetaspaceSize=128m",
  "-XX:ReservedCodeCacheSize=32m", "-Xss256k",
  "-XX:+UseCompressedOops", "-XX:+UseCompressedClassPointers",
  "-XX:+UseStringDeduplication", "-XX:+OptimizeStringConcat",
  "-XX:+AlwaysPreTouch", "-Djava.security.egd=file:/dev/./urandom",
  "-Dspring.jmx.enabled=false",
  "-jar", "/blog.jar"]
```

> 💡 **GraalVM vs HotSpot**：GraalVM 使用 Graal JIT 编译器替代 C2 编译器，性能提升 10-20%，内存节省约 19.5%。但需要预热 10-30 分钟才能体现优化效果。

### 4.6 构建并启动容器

```shell
cd /opt/aurora/app
docker build -t aurora-springboot:latest .
docker run -d --name aurora-springboot \
  --network aurora_aurora-network \
  -p 8080:8080 \
  --restart=always \
  aurora-springboot:latest
```

### 4.7 查看启动日志

```shell
docker logs -f aurora-springboot
```

### 4.8 访问测试

在浏览器访问：`http://服务器IP:8080`

## 🌐 6. Nginx配置

### 6.1 HTTP方式

在`/opt/aurora/nginx/`目录下创建`nginx.conf`：

```nginx
events {
    worker_connections  1024;
}

http {
    include       mime.types;
    default_type  application/octet-stream;
    sendfile        on;
    keepalive_timeout  65;

    client_max_body_size     50m;
    client_body_buffer_size  10m;
    client_header_timeout    1m;
    client_body_timeout      1m;

    gzip on;
    gzip_min_length  1k;
    gzip_buffers     4 16k;
    gzip_comp_level  4;
    gzip_types text/plain application/javascript application/x-javascript text/css application/xml text/javascript application/x-httpd-php image/jpeg image/gif image/png;
    gzip_vary on;

    server {
        listen       80;
        server_name  前台域名;

        location / {
            root   /usr/share/nginx/blog;
            index  index.html index.htm;
            try_files $uri $uri/ /index.html;
        }

        location ^~ /api/ {
            proxy_pass http://aurora-springboot:8080/;
            proxy_set_header   Host             $host;
            proxy_set_header   X-Real-IP        $remote_addr;
            proxy_set_header   X-Forwarded-For  $proxy_add_x_forwarded_for;
        }
    }

    server {
        listen       80;
        server_name  后台域名;

        location / {
            root   /usr/share/nginx/admin;
            index  index.html index.htm;
            try_files $uri $uri/ /index.html;
        }

        location ^~ /api/ {
            proxy_pass http://aurora-springboot:8080/;
            proxy_set_header   Host             $host;
            proxy_set_header   X-Real-IP        $remote_addr;
            proxy_set_header   X-Forwarded-For  $proxy_add_x_forwarded_for;
        }
    }
}
```

### 6.2 HTTPS方式

```nginx
events {
    worker_connections 1024;
}

http {
    include       mime.types;
    default_type  application/octet-stream;
    sendfile        on;
    keepalive_timeout  65;

    client_max_body_size     50m;
    client_body_buffer_size  10m;
    client_header_timeout    1m;
    client_body_timeout      1m;

    gzip on;
    gzip_min_length 1k;
    gzip_buffers     4 16k;
    gzip_comp_level 4;
    gzip_types text/plain application/javascript application/x-javascript text/css application/xml text/javascript application/x-httpd-php image/jpeg image/gif image/png;
    gzip_vary on;

    # SSL配置
    ssl_session_timeout 5m;
    ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:ECDHE:ECDH:AES:HIGH:!NULL:!aNULL:!MD5:!ADH:!RC4;
    ssl_protocols TLSv1.1 TLSv1.2 TLSv1.3;
    ssl_prefer_server_ciphers on;

    # HTTP到HTTPS重定向
    server {
        listen       80;
        server_name  前台域名 后台域名 minio域名;
        return 301 https://$host$request_uri;
    }

    # 前台配置
    server {
        listen  443 ssl;
        server_name  前台域名;

        ssl_certificate /etc/nginx/cert/前台域名_bundle.pem;
        ssl_certificate_key /etc/nginx/cert/前台域名.key;

        location / {
            root   /usr/share/nginx/blog;
            index  index.html index.htm;
            try_files $uri $uri/ /index.html;
        }

        location /api/ {
            proxy_pass http://aurora-springboot:8080/;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_connect_timeout 30s;
            proxy_send_timeout 30s;
            proxy_read_timeout 30s;
            proxy_intercept_errors on;
        }
    }

    # 后台配置
    server {
        listen  443 ssl;
        server_name  后台域名;

        ssl_certificate /etc/nginx/cert/后台域名_bundle.pem;
        ssl_certificate_key /etc/nginx/cert/后台域名.key;

        location / {
            root   /usr/share/nginx/admin;
            index  index.html index.htm;
            try_files $uri $uri/ /index.html;
        }

        location /api/ {
            proxy_pass http://aurora-springboot:8080/;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_connect_timeout 30s;
            proxy_send_timeout 30s;
            proxy_read_timeout 30s;
            proxy_intercept_errors on;
        }
    }

    # MinIO配置
    server {
        listen  443 ssl;
        server_name  minio域名;

        ssl_certificate /etc/nginx/cert/minio域名_bundle.pem;
        ssl_certificate_key /etc/nginx/cert/minio域名.key;

        location / {
            proxy_pass http://aurora-minio:9000/aurora/;
            proxy_set_header Host $proxy_host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_connect_timeout 30s;
            proxy_send_timeout 30s;
            proxy_read_timeout 30s;
            proxy_intercept_errors on;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
        }
    }
}
```

### 7.3 内存优化要点

1. **SpringBoot JVM极限优化**：64MB-128MB堆内存 + SerialGC (JDK 25)
2. **Elasticsearch优化**：64MB-160MB堆内存 + G1GC + IK分词器 + 断路器调优
3. **MySQL优化**：连接池30、缓冲池96M、表缓存100等极限调优
4. **Redis优化**：限制24MB内存，禁用持久化
5. **GraalVM 优化**（可选）：使用 Graal JIT 编译器，内存额外节省 19.5%
6. **整体效果**：
   - HotSpot JVM：总内存占用约 **1.51GB**，内存占用率约 **72.7%**
   - GraalVM JIT：总内存占用约 **1.44GB**，内存占用率约 **69.5%**

## 🚀 8. GraalVM JIT 模式迁移

### 8.1 为什么选择 GraalVM

| 优势 | 说明 |
|:-----|:-----|
| ✅ 零代码修改 | 完全兼容现有代码，迁移成本为 0 |
| ✅ 性能提升 | Graal JIT 编译器优化，预期 10-20% 性能提升 |
| ✅ 内存优化 | 更高效的内存管理，内存节省约 **19.5%** |
| ✅ 启动加速 | 优化的类加载机制，启动时间持平或略快 |
| ✅ 向后兼容 | 完全兼容 HotSpot JVM，无兼容性风险 |

### 8.2 性能对比实测

| 指标 | HotSpot JVM | GraalVM JIT | 改善幅度 |
|:------|:-----------:|:------------:|:--------:|
| **SpringBoot 内存** | 342.8 MiB | 275.9 MiB | ⬇️ **-19.5%** |
| **总内存占用** | 1539.44 MiB | 1472.55 MiB | ⬇️ -4.3% |
| **内存占用率** | 72.7% | 69.5% | ⬇️ -3.2% |
| **JIT 编译器** | C2 Compiler | Graal JIT | ⭐ 核心差异 |

### 8.3 快速迁移命令

```bash
# 1. 停止现有服务
docker stop aurora-springboot && docker rm aurora-springboot

# 2. 使用 GraalVM 启动服务
docker run -d --name aurora-springboot --network aurora_aurora-network -p 8080:8080 \
  -v /opt/aurora/app/aurora-springboot-0.0.1.jar:/app/blog.jar \
  -e SPRING_PROFILES_ACTIVE=prod -e TZ=Asia/Shanghai \
  -e SPRING_DATASOURCE_URL="jdbc:mysql://aurora-mysql:3306/aurora?serverTimezone=Asia/Shanghai&allowMultiQueries=true&useSSL=false&allowPublicKeyRetrieval=true" \
  -e SPRING_DATA_REDIS_HOST=aurora-redis -e SPRING_RABBITMQ_HOST=aurora-rabbitmq \
  -e ELASTICSEARCH_REST_URIS=http://aurora-elasticsearch:9200 \
  -e UPLOAD_MINIO_ENDPOINT=http://aurora-minio:9000/ \
  --restart=always ghcr.io/graalvm/jdk-community:25 \
  java -Xms64m -Xmx128m -XX:+UseSerialGC -XX:MetaspaceSize=64m -XX:MaxMetaspaceSize=128m \
       -XX:ReservedCodeCacheSize=32m -Xss256k -XX:+UseCompressedOops -XX:+UseCompressedClassPointers \
       -XX:+UseStringDeduplication -XX:+OptimizeStringConcat \
       -Djava.security.egd=file:/dev/./urandom -Dspring.jmx.enabled=false -jar /app/blog.jar

# 3. 查看日志
docker logs -f aurora-springboot
```

### 8.4 GraalVM 自动启用的优化

| 优化 | 说明 |
|:-----|:-----|
| ✅ **Graal JIT 编译器** | 替代 C2 编译器，性能提升 10-20% |
| ✅ **更激进的内联** | 方法调用优化，减少 5-10% 开销 |
| ✅ **更好的逃逸分析** | 栈上分配优化，减少堆压力 |
| ✅ **优化的死代码消除** | 减少无效代码，提升 CPU 效率 |

### 8.5 注意事项

| 序号 | 注意事项 |
|:----:|:---------|
| 1 | **预热期**：GraalVM 需要运行 10-30 分钟才能体现优化效果 |
| 2 | **镜像大小**：GraalVM 官方没有 Alpine 版本，镜像约 500MB（比 Alpine 大约 400MB） |
| 3 | **回滚方案**：保留原配置，可快速回滚到 HotSpot JVM |
| 4 | **生产部署**：建议先在测试环境验证，再灰度发布到生产 |

## ❓ 9. 常见问题

### 9.1 Docker相关

**容器一直重启**

```bash
docker logs 容器名称
```

**内存不足导致OOM**

- 检查docker-compose.yml中的内存限制配置
- 参考[内存优化总结](./内存优化总结.md)进行调整

**网络连接问题**

```bash
docker network ls
docker network inspect aurora_aurora-network
```

### 9.2 应用相关

**SpringBoot启动失败**

- 检查JVM参数是否正确
- 确认数据库、Redis等连接配置
- 查看日志：`docker logs aurora-springboot`

**Elasticsearch无法启动**

- 确认JVM内存设置（不低于64MB）
- 检查数据目录权限
- 查看日志：`docker logs aurora-elasticsearch`

**数据库连接失败**

- 确认数据库密码正确
- 检查MySQL连接数限制
- 验证网络连通性

### 9.3 其他问题

**如何修改网站基本信息**

打开后台管理系统，找到【系统管理 → 网站管理】，在此修改。

**ES安全认证**

当前配置已禁用ES安全认证（`xpack.security.enabled: "false"`），如需启用请修改docker-compose.yml。
